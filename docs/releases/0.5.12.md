# Helm 0.5.12

Helm 0.5.12 tightens the risk controls introduced in 0.5.11 by making the most judgment-heavy guardrail tunable and by improving snapshot resume context.

## Highlights

- `skill_relevance` blocking thresholds now live in `references/adaptive_harness_policy.json`
- operators can relax or tighten `skill_relevance_min_score`, `skill_relevance_block_verdicts`, and `skill_relevance_warn_verdicts`
- state snapshots now include harness routing metadata, skill relevance details, route decisions, and evidence presence
- snapshot tests now cover harness metadata so resume artifacts stay useful as the harness evolves

## Why this release matters

Skill relevance scoring is intentionally heuristic.
Hard-coding the block policy would make false positives harder to correct in real deployments.

This release keeps the guardrail active by default while allowing each workspace to tune how aggressively it blocks weak or poor skill matches.
It also makes state snapshots more useful for real continuation by carrying the execution contract signals that explain why a task was routed a certain way.

This is a patch release focused on reducing operational false positives and improving resume quality.
