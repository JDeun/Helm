# Helm v0.6.6 — Packaged References And Dogfooding Boundary

**Date:** 2026-04-27

Helm 0.6.6 fixes installed-package reference file reliability and promotes
portable operations improvements from OpenClaw dogfooding without importing
private workspace assumptions.

## Added

- product definition, module split, and dogfooding-boundary docs that define
  what belongs in public Helm versus a private OpenClaw-style overlay
- `scripts/state_snapshot.py` CLI for inspecting the latest Helm task state
  snapshot from a workspace
- `snapshot_payload(...)` helper for portable snapshot retrieval

## Changed

- packaged `references/*.json` and `references/*.md` so installed `helm init`
  can copy required reference files
- README and README.ko now point at v0.6.6 and link the public/private boundary
  docs
- README custom installer examples now wrap long commands for cleaner GitHub
  rendering
- onboarding, integration, context hydration, finalization, and knowledge-contract
  docs avoid repeating the same setup and promotion-policy details
- `ops_db` now shares JSONL parsing across task and command logs, indexes
  `command_index(task_id)`, accepts `guard.evaluated_at`, and verifies drift
  against latest task state rather than raw line count
- `run_with_profile` now centralizes guard fallback and blocked-task recording,
  and indexes task-ledger appends in one best-effort path
- state snapshots include `memory_capture.touched_paths` when present

## Excluded By Design

- OpenClaw-specific model fallback chains
- `~/.openclaw` hardcoded paths
- private Google/OAuth/Telegram/n8n/Obsidian automation
- local OpenClaw runtime patch wrappers
