# Helm 0.5.9

Helm 0.5.9 is a hardening patch release focused on keeping operator-facing inspection commands usable even when parts of the saved state are malformed.

## Highlights

- fixed `command_log_report.py` and `task_ledger_report.py` so malformed JSONL rows are skipped instead of aborting the whole report
- fixed `ops_daily_report.py` so malformed checkpoint indexes or draft assessment payloads degrade to empty sections instead of crashing inspection output
- expanded regression coverage around malformed-state handling in report and inspection paths

## Why this release matters

Inspection tools are most important when the workspace is already in a degraded state.
If a report command aborts because one JSON line or one assessment file is damaged, the operator loses visibility exactly when they need it.

This release narrows that failure mode:

- broken rows are treated as localized damage
- report commands continue to return the remaining valid state
- the operator can still inspect recent tasks, commands, checkpoints, and assessments without first hand-repairing every artifact

This is still a patch release.
The focus is report resilience and safer degradation under imperfect state.
