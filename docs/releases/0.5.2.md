# Helm 0.5.2

Helm 0.5.2 tightens the governance model around two practical issues that show up in real workspaces: not every disconnected durable file is real breakage, and not every draft skill should treat all source material as equal.

## Highlights

- clarified the operating rule that diagnostics should distinguish real breakage from intentional support artifacts such as projections, capture records, and alias stubs
- updated skill-quality guidance so source priority is treated as part of the visible skill contract when workflows can draw from multiple evidence sources
- refreshed the skill-capture template and draft review checklist so new skills start with stronger input and decision discipline

## Why this release matters

Helm already treated execution completion and operational completion as different boundaries. This release sharpens the same idea for inspection and skill quality:

- operators should not confuse support artifacts with defects
- skills should not treat weak and strong source material as interchangeable when the workflow depends on evidence quality

The result is a small patch release, but it improves how future workspaces explain diagnostics, capture policy, and draft-skill quality.
