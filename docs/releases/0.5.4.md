# Helm 0.5.4

Helm 0.5.4 makes local file intake a first-class contract and adds clearer inspection surfaces for recent task capability state.

## Highlights

- added `file_intake` evidence contracts to adaptive harness manifests, harness recording paths, and task-ledger summaries
- expanded manifest-quality auditing so skills that require file intake must explain that boundary in `SKILL.md`
- added `helm run-contract` and `helm capability-diff` so operators can inspect recent execution contract state without reading raw ledger rows
- added a dedicated knowledge contract doc to separate transcript state, durable memory, workflow artifacts, and promoted skill knowledge

## Why this release matters

Previous Helm releases made browser work and retrieval escalation inspectable. This release closes a similar gap for local-file workflows:

- the operator should be able to see what file type was claimed versus detected
- file-driven skills should expose whether the workflow was safe to parse or should have been routed elsewhere
- capability drift between recent tasks should be visible without manual ledger diffing

This is still a patch release, but it makes file-oriented workflows and recent-run inspection materially easier to govern in long-lived agent workspaces.
