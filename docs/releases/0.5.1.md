# Helm 0.5.1

Helm 0.5.1 cleans up the public repository boundary after the 0.5.0 governance expansion.

The previous release made `SKILL.md` part of the operator-facing contract model. This patch release makes the repository consistent with that idea: private skill assets no longer live at the repository root, public docs point at generalized demo assets, and the validation story is covered by broader automated tests.

## Highlights

- removed repository-root personal skill contracts so the public repo does not ship private workflow assets
- switched documentation and harness examples to the generalized `router-context-demo` draft in `examples/demo-workspace`
- kept the repository root skill registry empty by default while preserving a realistic demo workspace for validation
- expanded automated tests to cover both manifest-quality helpers and CLI validation flows

## Recommended validation flow

```bash
python3 -m pip install --user --no-build-isolation .
helm --help
python3 scripts/run_with_profile.py validate-manifests --json
python3 scripts/run_with_profile.py audit-manifest-quality --json
HELM_WORKSPACE=examples/demo-workspace python3 scripts/run_with_profile.py validate-manifests --json
HELM_WORKSPACE=examples/demo-workspace python3 scripts/run_with_profile.py audit-manifest-quality --json
helm validate --path examples/demo-workspace
python3 -m unittest discover -s tests -p 'test_*.py'
```

## Notes

- This is a public-repo hygiene release, not a feature reset.
- Helm still expects users to define their own skill contracts in their own workspaces.
- The demo workspace remains the reference place for portable, publishable sample skill governance assets.
