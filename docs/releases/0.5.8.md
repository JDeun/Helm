# Helm 0.5.8

Helm 0.5.8 is a hardening patch release focused on audit-path correctness, safer restoration boundaries, and more reliable evidence capture.

## Highlights

- fixed adaptive harness hydration command generation so empty `include` lists no longer produce invalid `ops_memory_query` invocations
- fixed route-decision tool inference for nested shell commands that prepend environment variables before the real runner
- fixed context-source loading so corrupted or partial `.helm/context_sources.json` files do not break local context inspection
- fixed checkpoint restore so symlink and hardlink members inside archives are rejected instead of being restored into the workspace
- expanded audit coverage and tests around workspace detection, checkpoint restore, hydration commands, and context fallback behavior

## Why this release matters

The main risk here was not only obvious crashes.
It was silent degradation in the paths operators rely on when they need trust the most:

- hydration commands could be generated in a shape that the downstream parser could not actually execute
- route inference could validate the wrong tool when a nested shell command used environment-variable prefixes
- a damaged context-source file could block all adopted-context inspection even though local state was still readable
- checkpoint restore could accept link members that do not belong in a conservative workspace recovery path

This release tightens those edges so Helm behaves more predictably under audit, recovery, and inspection-heavy workflows.

This is still a patch release.
The focus is correctness, safer failure modes, and operator confidence.
