# Helm 0.5.5

Helm 0.5.5 turns memory policy into a more inspectable runtime surface instead of leaving it only in prose.

## Highlights

- added a runtime-neutral memory operations policy for crystallization, confidence/recency, supersession, review flags, and scope boundaries
- expanded Helm's own `memory_capture` payload so recent-state and pending-capture inspection can show confidence, retention, review, and supersession hints
- refreshed release metadata and release checklist for the new patch version

## Why this release matters

Previous Helm releases could tell operators whether durable capture was pending.
This release makes the next question more inspectable:

- how trustworthy is the captured claim
- whether the run looks like it replaced older state
- whether the output should stay episodic or become a stronger reusable asset
- whether the runtime should require review before broader promotion

This is still a patch release.
The focus is not bigger automation.
The focus is clearer policy visibility for long-lived agent workspaces.
